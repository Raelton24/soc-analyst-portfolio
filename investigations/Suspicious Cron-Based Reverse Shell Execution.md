## Executive Summary

During a review of Linux cron logs collected through rsyslog, a suspicious command executed by the root account was identified. The command launches an interactive Bash shell and redirects its input and output over a TCP connection to an external host (`34.253.159.159`) on port `9999`.

The observed behavior strongly resembles a reverse shell, a technique commonly associated with post-exploitation, persistence, and unauthorized remote command execution.

Although the logs confirm execution of the command, they do not establish whether the outbound connection was successfully completed. Additional evidence from network, firewall, or endpoint telemetry would be required to determine whether a remote session was established.

---

## Investigation Scope

This investigation focuses on analyzing Linux cron logs collected through `rsyslog` to identify suspicious scheduled task activity.

## Technical Analysis

The command launches an interactive Bash shell and redirects its standard input, output, and error streams over a TCP connection to an external host.

Key observations include:

- Executed with root privileges.
- Initiated from a scheduled cron task.
- Establishes outbound communication to an external IP.
- Provides interactive shell capability if the remote endpoint is listening.

These characteristics are commonly associated with reverse-shell behavior used during post-exploitation.

## Risk Assessment

**Severity:** High

Execution of a reverse shell as the root user presents a significant security risk due to the potential for full system compromise and persistent unauthorized access.

---

## Analyst Confidence

**Confidence Level:** Medium

The available evidence confirms execution of the reverse-shell command. However, additional telemetry would be required to determine whether the remote connection was successfully established.

## Recommended Next Steps

- Review firewall logs for outbound traffic to `34.253.159.159`.
- Correlate activity in Wazuh or another SIEM.
- Examine process creation events around the execution time.
- Review authentication logs for potential initial access.
- Investigate additional persistence mechanisms.
- Determine whether other hosts communicated with the same IP.

---

## Lessons Learned

- Cron logs can reveal unauthorized scheduled task execution.
- rsyslog enables targeted log collection for investigation.
- Reverse shells can often be identified directly from command-line artifacts.
- Log analysis becomes significantly more valuable when correlated with network telemetry.
